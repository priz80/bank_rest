## 🏦 **Система управления банковскими картами**
> Описание проекта по ТЗ

---

## 📋 Общее описание

Backend-приложение на Java (Spring Boot 3) для управления банковскими картами с поддержкой ролей, JWT-аутентификации, шифрования данных и перевода средств между своими картами.

Приложение соответствует современным стандартам безопасности, масштабируемости и удобства разработки.

---

## 💳 Основные функции

### Пользователь (USER)
- Просмотр своих карт (с пагинацией и фильтрацией)
- Создание новой карты
- Блокировка и активация карт
- Перевод средств между своими картами
- Просмотр баланса
- Маскировка номера карты: `**** **** **** 1234`

### Администратор (ADMIN)
- Полный доступ ко всем картам системы
- Управление пользователями: создание, блокировка, удаление
- Просмотр всех пользователей и их карт
- Изменение статусов

---

## 🔐 Безопасность

- JWT-аутентификация с токеном в заголовке `Authorization`
- Ролевой доступ:
  - `USER` — только свои данные
  - `ADMIN` — полный доступ
- Шифрование номера карты и CVV при сохранении в БД (AES-256)
- Маскирование номера при отображении
- Валидация входных данных и понятные сообщения об ошибках

---

## 🗄 Работа с базой данных

- **PostgreSQL** — основная СУБД
- **Liquibase** — управление миграциями (`src/main/resources/db/changelog`)
- Автоматическое применение схемы при запуске
- Поддержка пагинации и сортировки через `Pageable`

## 🌐 REST API

### Эндпоинты

#### Авторизация
- `POST /api/auth/login` — Получение JWT токена

#### Карты (USER)
- `GET /api/cards` — Получить свои карты (с пагинацией: `page`, `size`, `sort`)
- `POST /api/cards` — Создать новую карту (автогенерация номера, CVV, срока)
- `GET /api/cards/{id}` — Получить карту по ID
- `POST /api/cards/{id}/block` — Заблокировать карту
- `POST /api/cards/{id}/activate` — Активировать карту
- `DELETE /api/cards/{id}` — Удалить карту (если не активна и баланс = 0)
- `POST /api/cards/transfers` — Перевод средств между своими картами

#### Администрирование (ADMIN)
- `GET /api/admin/users` — Все пользователи (пагинация, сортировка)
- `GET /api/admin/users/{id}` — Пользователь по ID
- `POST /api/admin/users` — Создать пользователя
- `PUT /api/admin/users/{id}/status` — Изменить статус пользователя
- `DELETE /api/admin/users/{id}` — Удалить пользователя (только если DELETED и нет карт)
- `GET /api/admin/cards` — Все карты в системе

## 🚀 Запуск и развёртывание

### Требования
- Java 17
- Docker & Docker Compose

### Команды

```bash
# Сборка проекта
mvn clean package -DskipTests

# Остановка и удаление старых контейнеров
docker-compose down -v

# Сборка и запуск
docker-compose up --build
```

После запуска:
- Приложение: [http://localhost:8080](http://localhost:8080)

## ✅ Особенности реализации

| Функция | Реализация |
|--------|-----------|
| Генерация номера карты | Алгоритм Луна, длина 16 цифр |
| Генерация CVV | Случайные 3 цифры |
| Срок действия | +4 года от даты создания |
| Шифрование | AES-256 через `@Convert` и `AttributeConverter` |
| Автообновление статуса | `@Scheduled` — проверка просроченных карт ежедневно |
| Маскировка номера | В DTO — `1234 **** **** 5678` |
